# Contributing to Winx

Thank you for your interest in contributing to Winx! Your contributions help make this project better and more useful for the community.

This document provides an overview of how to get involved, where to find guidelines, and how to submit your changes.

> [!IMPORTANT]
> Before contributing, please review the detailed guidelines on the project’s structure and workflow in the contributing documentation:
>
> - [docs/contributing](docs/contributing/README.md) — Overview and index of contributing resources

---

## How to Contribute

### Reporting Issues

- Use the GitHub Issues tab to report bugs or suggest new features.
- Please provide detailed descriptions, reproduction steps, and relevant environment information.

### Submitting Pull Requests

- Fork the repository and create a feature branch.
- Write clear, concise commit messages.
- Ensure your code follows PowerShell best practices.
- Add or update Pester tests where applicable.
- Test your changes thoroughly before submitting.
- Submit a pull request with a detailed explanation of your changes.

### Code Style and Quality

- Follow the project's directory and coding standards outlined in the contributing documentation.
- Maintain consistent naming conventions and module structure.

---

## Getting Help

If you have questions or need assistance, feel free to open an issue or contact the maintainers.
